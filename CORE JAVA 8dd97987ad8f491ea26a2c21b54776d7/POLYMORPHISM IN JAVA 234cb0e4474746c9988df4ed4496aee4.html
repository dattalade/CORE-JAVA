POLYMORPHISM IN JAVA

Polymorphism in Java

  • Poly means multiple, morphism means forms, so it is called as Multiple Forms
  • We can perform polymorphism in two ways :
    1. Method Overloading
    1. Method Overriding

      Method Overloading

      • If a class contains multiple methods of symmetric name, but vary either in parameter size or parameter datatypes is known as

        Method Overloading

      • It is also known as Compile Time Polymorphism

      Example_1 (OverLoad.java) →

      package com.overloading;
import java.util.*;

class Addition
{
	public int add(int a,int b)
	{
		return a+b;
	}

	public int add(int a,int b, int c)
	{
		return a+b;
	}
}

public class Overload
{
	public static void main(String[] args)
	{
		Addition a = new Addition();
		System.out.print(a.add(1,2) + " ");
		System.out.print(a.add(1,2,3));
	}
}

      Output →

      3 6

      Method Overriding

      • Having same methods in both subclass & superclass is called Method Over-riding
        • we already discussed in super() keyword
      • Method overriding is used for runtime polymorphism

        Rules for Java Method Overriding

        1. The method must have the same name as in the parent class
        1. The method must have the same parameter as in the parent class.
        1. There must be an IS-A relationship (inheritance).

  • There are two types of Polymorphism :
    1. Compile-time Polymorphism → Overloading the static methods methods are called as Compile-time ploymorphism
    1. Run-time Polymorphism

      Run-time Polymorphism

      • It is a process in which a call to an overridden method is resolved at runtime rather than compile-time.
      • Runtime polymorphism is also known as Dynamic Method Dispatch
      • In this process, an overriding method is called through the reference variable of a superclass / parent class / base class
      Note : If the reference variable of Parent class refers to the object of Child class, it is known as upcasting.

      Example_1 (Main.java) →

      package com.runtime_polymorphism;
import java.util.*;

class Bike
{  
	void run()
	{
		System.out.println("running");
	}  
}  

class Splendor extends Bike
{  
	void run()
	{
		System.out.println("Splendor running safely with 60km");
	} 
}

class Yamaha extends Bike
{  
	void run()
	{
		System.out.println("Yamaha running safely with 40km");
	} 
}

public class Main
{
	public static void main(String args[])
	{  
    Bike b1 = new Splendor();    //upcasting  
    b1.run();  
		Bike b2 = new Yamaha();    //upcasting  
    b2.run();  
  }  
}

      Output →

      Splendor running safely with 60km

      Yamaha running safely with 40km

Static Binding

  • When type of the object is determined at compiled time(by the compiler), it is known as static binding.
  • If there is any private, final or static method in a class, there is static binding.

Example_1 (Dog.java) →

package com.staticbinding;
import java.util.*;

public class Dog
{  
	private void eat()
	{
		System.out.println("dog is eating...");
	}  
  
	public static void main(String args[])
	{  
		Dog d1=new Dog();  
		d1.eat();  
	}
}

Output →

dog is eating. . .
