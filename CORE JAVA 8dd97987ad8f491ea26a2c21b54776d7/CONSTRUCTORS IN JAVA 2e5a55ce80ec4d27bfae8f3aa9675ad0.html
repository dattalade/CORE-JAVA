CONSTRUCTORS IN JAVA

Constructors in Java

  • Constructor is a block of code to perform specific task which is similar to method, but it is called as Special Method
  • Constructor is invoked when an instance(object) of the class is created.
  • At the time of calling constructor, memory for the object is allocated in the memory.
  • It there is no constructor available, Java Compiler invokes Default Constructor while creating an Object.
  • We can use access modifiers for constructor which controls Object Creation

    Rules for Creating Constructor

    1. The Constructor name must be same as Class Name
    1. A Constructor must not have a explicit return type
    1. It cannot be static because Constructor is invoked by creating Object
    1. It cannot be final because internally Constructor is final
    1. It cannot be abstract, synchronized
  • Types of Java Constructor :
    1. Default Constructor
    1. Parameterized Constructor

    Default Constructor

    • It is known as no-argument (not parameterized) Constructor
    • We will not pass any arguments while creating an Object

    Syntax →

    className()
{
	//code
}

    Example_1 (Bike.java) →

    package com.constructors;
import java.util.*;

public class Bike
{
	Bike()
	{
		System.out.println("Bike '" + this + "' is created");
	}
	public static void main(String[] args)
	{
		Bike b = new Bike();
	}
}

    Output →

    Bike 'Bike@12345t' is created

    If there is no constructor in the class →

    Example_2 (Bike.java) →

    package com.constructors;
import java.util.*;

public class Bike
{
	int id;
	
	public static void main(String[] args)
	{
		Bike b = new Bike();
		System.out.println(b.id);
	}
}

    Output →

    0

    Internal Program →

    Bike b = new Bike() → Compiler → Bike()
    {
    this.id = 0;
    }

    Parameterized Constructor

    • It is known as argument Constructor
    • We will pass one or many arguments while creating an Object

    Syntax →

    className(/* parameter list */)
{
	//code
}

    Example_1 (Bike.java) →

    package com.constructors;
import java.util.*;

public class Bike
{
	int id;
	Bike(int id)
	{
		this.id = id;
		System.out.println(this.id);
	}
	public static void main(String[] args)
	{
		Bike b = new Bike(201);
	}
}

    Output →

    201

Constructor Overloading in Java

  • Having more than one Constructor with different size of parameters or same size of parameters (but vary in dataType) is called Constructor Overloading.
  • Each constructor will perform it's own task.

Example_1 (Bike.java) →

package com.constructors;
import java.util.*;

public class Bike
{
	Bike(int id)
	{
		System.out.println(id);
	}
	Bike(String name)
	{
		System.out.println(name);
	}
	Bike(int id, String name)
	{
		System.out.println(id + " " + name);
	}
	public static void main(String[] args)
	{
		Bike b1 = new Bike(201);
		Bike b2 = new Bike("datta");
		Bike b3 = new Bike(201, "datta");
	}
}

Output →

201

datta

201 datta

Drawbacks of Constructor Overloading

  • Every Constructor is invoked with different Objects not with same Objects
  • So that , data is divided into different objects which is diffucult while performing any large application

Constructor Chaining in Java

  • To overcome the limitation of Constructor Overloading , Constructor Chaining is presented
  • Calling one Constructor to another Constructor called Constructor Chaining
  • Another Constructors can be called using this keyword

Example_1 (Bike.java) →

package com.constructors.chaining;
import java.util.*;

public class Bike
{
	int id;
	String name;

	Bike(int id, String name)
	{
		this.id = id;
		this(name);
	}
	Bike(String name)
	{
		this.name = name;
	}

	public static void main(String[] args)
	{
		Bike b1 = new Bike(5, "splendor");
		System.out.println(b1.id + " " + b1.name);
	}
}

Output →

5 splendor

Behaviour of Constructors while specifying Access Modifiers

  • Default Access Modifier for any constructor is default
  • If the public is the modifier , the object can be created at any package or class
  • If the private is the modifier , the object can be created at it's own class
  • If the default is the modifier , the object can be created at it's specific package
  • If the protected is the modifier , the object can be created at it's derived class / sub class

Example_1 (Main.java) →

package com.constructors.chaining;
import java.util.*;

class Bike
{
	int id;
	String name;

	private Bike(int id)
	{
		this.id = id;
	}
	Bike(String name)
	{
		this.name = name;
	}
}

public class Main
{
	public static void main(String[] args)
	{
		Bike b1 = new Bike(31692);
		Bike b2 = new Bike("datta");
	}
}

Output →

error: incompatible types: int cannot be converted to String

Reason →

One of the Single Parameterized Constructor is hiding which have as an int as parameter and other Single Parameterized Constructor having String as parameter is visible , So when Bike b1 object is creating it is observing only Single Parameterized Constructor having String as parameter where int as parameter is hiding.

Example_2 (Main.java) →

package com.constructors.chaining;
import java.util.*;

class Bike
{
	int id;

	private Bike(int id)
	{
		this.id = id;
	}

public class Main
{
	public static void main(String[] args)

Reason →

The Constructor is private accessed which only it’s own class can observe and access

Differences between Constructor & Method

Constructor Method
It is used to initialize the state of the ObjectIt is used for functionality or behaviour of Object
It must not have return typeIt must have return type. i.e int, float, void , etc…
It is invoked implicitlyIt is called explicitly
Default Constructor is provided by Compiler, if a constructor is not definedThe method is not provided by Compiler, our own methods
The Constructor name must be the same as Class NameThe method name can be of any name
