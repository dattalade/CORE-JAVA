METHODS IN JAVA

Methods in Java

  • It is the block of code or collection of instructions which performs specific task
  • It provides code reusability (write method once & and use it multiple times)
  • It also provides the easy modification and readability of code, just by adding or removing a chunk of code.
  • The method is executed only when we call or invoke it.
  • There are two ways to create methods :
    1. Object Methods
    1. Static Methods

    Object Methods

    Syntax →

    accessSpecifier returnType methodName(parameterList (optional))
{
	//code	
}

    • Access specifier or Access modifier →
      • It is the access type of the method. It specifies the visibility of the method.
      • Java provides four types of access specifier:
        1. public → The method is accessible by all classes when we use public specifier in our application.
        1. private → When we use a private access specifier, the method is accessible only in the classes in which it is defined.
        1. protected → When we use this access specifier, the method is accessible within the same package or subclasses in a different package.
        1. default → When we do not use any access specifier in the method declaration, Java uses default access specifier by default. It is visible only from the same package only.
        Access Modifierwithin classwithin packageoutside package by subclass onlyoutside package
        PrivateYESNONONO
        PublicYESYESYESYES
        ProtectedYESYESYESNO
        DefaultYESYESNONO
    • Return Type →
      • Return type is a data type that the method return a value.
      • The return type maybe primitive data type, object, collection, void, etc.
      • If the method does not return anything, we use void keyword.
    • Method Name →
      • It is a unique name that is used to define the name of a method.
      • It must be corresponding to the functionality of the method.
    • Parameter List →
      • It is the list of parameters separated by a comma and enclosed in the pair of parentheses.
      • It contains the data type and variable name. If the method has no parameter, left the parentheses blank.

    Example_1 (Hello.java) →

    package com.methods;
import java.util.*;

public class Hello
{
	String value;

	public void printValue(int name)
	{
		value = name;
		System.out.println(value);
	}
	public static void main(String[] args)
	{
		Hello h1 = new Hello();
		String name = "Hello";
		h1.printValue(name);
	}
}

    Output →

    Hello

    Note : We are in object class, So th access data members there is no need to use refernce variable

    Example_2 (Addition.java) →

    package com.methods;
import java.util.*;

public class Addition
{
	int a, b;

	public int addValues(int p, int q)
	{
		a = p;
		b = q;
		return a+b;
	}
	public static void main(String[] args)
	{
		Addition a1 = new Addition();
		int sum = h1.addValues(10,20);
		System.out.println(sum);
	}
}

    Output →

    30

    Example_2 (Main.java) → Calling a method from one Class to Another Class in 'Object- way'

    package com.methods;
import java.util.*;

class Factorial
{
	int number;
	public int factorial()
	{
		int mul = 1;
		for(int i=number;i>=2;i--)
			mul*=i;
		return mul;
	}
}
public class Main
{
	public static void main(String[] args)
	{
		Factorial f1 = new Factorial();
		Factorial f2 = new Factorial();
		f1.number = 4; f2.number = 5;
		System.out.println(f1.factorial() + " " + f2.factorial());
	}
}

    Output →

    24 120

    Static Methods

    • A method that has static keyword is known as static method.
    • A method that belongs to a class rather than an instance of a class is known as a static method.
      • In Object methods, the method is belong to each object

    Advantages of Static Methods →

    1. We can call the method without creating an object. It can access static data members and also change the value of it.
    1. Let us assume a Students Database of same University
      • Then, Every Student belongs to same university so we can use university variable as static

    Syntax →

    accessSpecifier static returnType methodName(parameterList (optional))
{
	//code	
}
								(or)
static accessSpecifier returnType methodName(parameterList (optional))
{
	//code	
}

    Example_1 (Addition.java) →

    package com.methods;
import java.util.*;

public class Addition
{
	public static int addValues(int p, int q)
	{
		return p+q;
	}
	public static void main(String[] args)
	{
		int sum = Addition.addValues(10,20);
		System.out.println(sum);
	}
}

    Output →

    30

    Example_2 (Main.java) → Calling a method from one Class to Another Class in 'Static way'

    package com.methods;
import java.util.*;

class Factorial
{
	public static int factorial(int number)
	{
		int mul = 1;
		for(int i=number;i>=2;i--)
			mul*=i;
		return mul;
	}
}
public class Main
{
	public static void main(String[] args)
	{
		int a = 3, b = 4, c = 5;
		int ta = Factorial.factorial(a);
		int tb = Factorial.factorial(b);
		int tc = Factorial.factorial(c);
		System.out.println(ta + " " + tb + " " + tc);
	}
}

    Output →

    6 24 120

  • There are two types types of Methods →
    1. User-Defined Methods (The above examples are user Defined Methods)
    1. Pre-Defined Methods (The method which we invoke without writing the block of code

this Keyword

  • While we call a method, to address the current object we use 'this' keyword

    (or)

  • When we have the Data Member & Parameter Variable name as same then we use this keyword

Example (Hello.java) →

package com.methods;
import java.util.*;

public class Hello
{
	int a;
	public void printIt(int a)
	{
		this.a = a;
	}

  public static void main(String[] args)
  {
  	Hello h1 = new Hello();
  	h1.printIt(3);
  	System.out.println(h1.a);
  }
}

Output →

3
