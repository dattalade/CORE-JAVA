EXCEPTIONS IN JAVA

What is an Exception?

  • In Java, an exception is an event which disrupts or exit from the flow of execution
  • The Exception is an Object which throws at runtime

What is Exception Handling?

  • Exception Handling is one of the most powerful mechanism in Java
  • This mechanism can handle runtime errors in Java, so that normal flow of execution can be maintained which a big adavantage
  • So this feautre refers to robust
  • Let us assume →
    statement 1;  
statement 2;  
statement 3;  
statement 4;  
statement 5;    //exception occurs  
statement 6;  
statement 7;  
statement 8;  
statement 9;  
statement 10;
    • Suppose there are 10 Statements in a program, but if a runtime error occured in statement 5, we can handle to run other 5 statements using Exception Handling

Hierarchy of Java Exception classes

  • java.lang is the package where Exception classes are available, also it is a package which is default imported by Java Intrepreter
  • java.lang.Throwable is the root of all Exceptions inherited by two subclasses
graph TD
  A[Throwable] --> B[Exception]
	A --> C[Error]
	B --> D[IOException]
	B --> E[SQLException]
	B --> G[RunTimeException]
	B --> F[ClassNotFoundException]
	C --> H[StackOverFlowError]
	C --> I[VirtualMachineError]
	C --> J[OutOfMemoryError]
	G --> K[ArithmeticException]
	G --> L[NullPointer Exception]
	G --> M[NumberFormat Exception]
	G --> N[IndexOutOfBounds Exception]
	N --> O[ArrayIndexOutOfBoundsException]
	N --> P[StringIndexOutOfBoundsException]

Java try-catch block

Syntax →

try
{    
	//code that may throw an exception    
}
catch(Exception_class_Name ref (or) Exception ref)
{
	//Handling the Exception Code
}

Internal Working →

  • The JVM firstly checks whether the exception is handled or not. If exception is not handled, JVM provides a default exception handler that performs the following tasks:
    • Prints out exception description.
    • Prints the stack trace (Hierarchy of methods where the exception occurred).
    • Causes the program to terminate.

Java Multiple Catch blocks

  • A try block can have one or more catch blocks. Each catch block will handle a different Exception.
  • So, if you want to perform the tasks based on an exceptions we use multiple catch blocks
Note : At a time only one exception occurs and at a time only one catch block is executed.
Note : • All catch blocks must be ordered from most specific to most general, i.e. catch for ArithmeticException must come before catch for Exception.

Syntax →

try
{    
	//code that may throw an exception    
}
catch(Exception_class_Name ref (or) Exception ref)
{
	//Handling the Exception Code
}
catch(Exception_class_Name ref (or) Exception ref)
{
	//Handling the Exception Code
}
-------------------
catch(Exception_class_Name ref (or) Exception ref)
{
	//Handling the Exception Code
}

Example_1 (MultipleCatch.java) →

package com.exceptions;
import java.util.*;

public class MultipleCatch
{
	public static void main(String[] args)
	{
		try
    {
	    int a[] = new int[5];    
      a[5] = 30/0;
    }
		catch(ArithmeticException ae)
    {
		  System.out.println("Can't Divide with Zero");
    }
		catch(ArrayIndexOutOfBoundsException ae)
    {
		  System.out.println("Length is out of boundary");
    }
		catch(Exception e)
    {
		  System.out.println("Parent Exception");
    }
    System.out.println("rest of the code...");
	}
}

Nested try blocks

  • In Java, using a try block inside a try block is called as Nested try blocks

Syntax →

try
{
	// First try code
	try
	{
		// Second try code
		-------------------
		try
		{
			// nth try Code
		}
		catch(Exception_class_Name ref (or) Exception ref)
		{
			//Handling the Exception Code
		}
	}
	catch(Exception_class_Name ref (or) Exception ref)
	{
		//Handling the Exception Code
	}
}

Java finally Block

  • Java finally block is always executed whether an exception is handled or not.

Flowchart of finally Block →

graph TD;
  A[Program_Code] --> B{Exception_Occured?}
	B -- NO --> C[Finally_Block_Executed]
	B -- YES --> D{Exception_Handled?}
	D -- NO --> C
	D -- YES --> C

Example_1 (FinallyBlock.java) →

package com.exceptions;
import java.util.*;

public class FinallyBlock
{
	public static void main(String[] args)

		{  
			System.out.println("NullPointer Exception");  
		}    
	  finally
		{  
			System.out.println("finally block is always executed");  
		}

	}
}

5

Java throw keyword

  • The Java throw keyword is used to throw an exception explicitly.
  • We can throw either checked or unchecked exception or User-Defined exceptions.

Syntax →

throw Exception_Class("errorMessage");

Example_1 (Exceptions.java) → Throwing Unchecked Exception

package com.exceptions;
import java.util.*;

public class Exceptions
{
	public static void main(String[] args)
	{
		int age = 17;
		if(age < 18)
		{
			throw new ArithmeticException("Person is not eligible to vote");
		}
		else
		{
			System.out.println("Person is eligible to vote");
		}
	}
}

Output →

Exception in thread "main" java.lang.ArithmeticException:/ Person is not eligible to vote

Note : throw keyword works only for UnChecked Exceptions (under our control, we can correct our code) and error like StackOverFlowError

Exception Propagation

  • If a method cannot handle an exception, then caller method tries to handle the exception, and so on is called Exception Propogation.

Example_1 (ExceptionPropagation.java) →

package com.exceptions;
import java.util.*;

public class ExceptionPropagation
{
	void m()
	{  
    int data=50/0;  
  }  

  void n()
	{  
    m();  
  }  

  void p()
	{  
		try
		{  
	    n();  
	  }
		catch(Exception e)
		{
			System.out.println("Propogation Performed");
		}  
  }

	public static void main(String[] args)
	{
		ExceptionPropagation ep = new ExceptionPropagation();
		ep.p();
	}
}

Propogation Performed

Exception Handling with Method Overriding

  • There are many rules while handling exception in Inheritance

Rule_1 : If the superclass method does not declare an exception, subclass overriding method cannot declare the checked exception.

Example_1 (Rule1.java) →

package com.exceptions;
import java.util.*;
import java.io.*;

class Rules
{
	public void addRule()
	{
		System.out.println("Added Rules");
	}
}

public class Rule1 extends Rules
{
	public void addRule() throws IOException
	{
		System.out.println("Added Rule1");
	}

	public static void main(String[] args)
	{
		Rule1 r1 = new Rule1();
		r1.addRule();
	}
}

Output →

error: addRu1e() in Main cannot override addRu1e() in Rules

public void addRu1e() throws java . io.IOException

^
overridden method does not throw IOException

Custom Exception

  • We can create our exceptions with our own class, this phenomenon is called as Custom Exception or User-Defined Exception
  • The need of this exception is o customize the exception according to requirements

Syntax →

public class <class_Name> extends Exception
{
	<class_Name>(String str)
	{
		super(str);
	}
}

Example_1 (CustomException.java) →

package com.exceptions;
import java.util.*;

class InvalidAgeException extends Exception
{
	InvalidAgeException(String str)
	{
		super(str);
	}
}

public class CustomException
{
	public void ageValid(int age) throws InvalidAgeException
	{
		if(age < 18)
			throw new InvalidAgeException("Your age is : " + age + ", where you are not eligible to vote");
	}

	public static void main(String[] args) throws InvalidAgeException
	{
		CustomException ce = new CustomException();
		ce.ageValid(17);
	}
}

Output →

Exception in thread "main" InvalidAgeException: Your age is : 17, where you are not eligible to vote
